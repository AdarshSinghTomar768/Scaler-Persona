Adarsh AI Persona Evals Report

The system was evaluated on voice performance, chat groundedness, failure handling, and production readiness.

Voice Quality

  • Latency: measured time to first spoken response after caller utterance end, with a target below 2 seconds.
  • Accuracy: compared spoken answers against the retrieved resume and repo evidence.
  • Task completion: checked whether the agent answered follow-ups, offered availability, and completed booking.

Chat Groundedness

  • Hallucination rate: sampled answers for unsupported claims across resume, project, and fit questions.
  • Retrieval quality: validated that the top chunks matched the intended source, especially for RAG and project prompts.
  • Citation usefulness: confirmed the UI showed enough source context for manual verification.

Failure Modes And Fixes

  • Resume retrieval was too coarse. Fix: added PDF extraction and tighter chunking.
  • Fit questions retrieved repo docs instead of resume facts. Fix: added query expansion and resume boosts.
  • Booking could look successful without provider credentials. Fix: return explicit missing_credentials errors.

Two More Weeks

  • Add full conversational tool calling so chat can gather booking info end to end without a separate form.
  • Add automated eval harnesses for latency, interruption handling, and retrieval precision at k.
  • Add deployment automation for Vercel or Fly.io plus bootstrap scripts for Vapi, Twilio, and Cal.com.
